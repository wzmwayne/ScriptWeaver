🦋 灵蝶 ScriptWeaver 功能演示

本页面仅展示如何声明灵蝶的各项能力，实际效果需要安装脚本并刷新。

1. 远程脚本加载

通过 meta 标签指定一个远程 JS 文件，灵蝶会自动拉取并执行。

<meta name="tm-dynamic-script" content="https://你的域名/script.js">
↑ 此处已声明一个示例 URL（实际不存在，仅做演示）。

2. 内嵌动态代码

使用自定义 script 标签直接嵌入 JS 或 JSON 指令，优先级高于远程。

<script type="x-tm-inline">
    document.body.style.borderLeft = '5px solid #3498db';
    console.log('内嵌代码已执行');
</script>

3. GM 权限声明

页面可告知灵蝶需要哪些高级权限，脚本会自动提示缺失项。

<meta name="scriptweaver-permissions" content="xhr,storage,notification">
↑ 已声明：跨域请求、本地存储、通知权限。查看右上角 Toast 提示。

4. 安全指令（强制模式）

将灵蝶设为模式 1，只解析 JSON 指令，杜绝任意 JS。适合静态内容注入。

<script type="x-tm-inline">
[
  {"op": "setText", "selector": "#instruction-demo", "text": "这是通过安全指令更新的文字"},
  {"op": "addClass", "selector": "#instruction-demo", "className": "demo-area"}
]
</script>
原始文字
↑ 如果灵蝶处于强制模式，上面文字会被替换。

5. 环境自检

检查浏览器是否支持灵蝶所需的基础能力。

💡 提示：以上所有功能均由 ScriptWeaver 脚本 驱动。
请确保已安装 Tampermonkey 并添加了最新版灵蝶脚本。
